# Sanity CMS Integration Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Initialize and configure Sanity Studio in `studio-mergex-console` and set up Next.js rewrites to serve Sanity at `mergex.in/console` (mapping to local port 3333 in development).

**Architecture:** Use a standalone Sanity Studio directory `studio-mergex-console` under the workspace root. We use Next.js rewrites in `next.config.ts` to reverse-proxy `/console/:path*` to the Sanity Studio (hosted on port 3333 during local development).

**Tech Stack:** Sanity CMS, Next.js 16, TypeScript, React 19.

#### [NEW] [studio-mergex-console/](file:///d:/web%20development/the-mergex-company/studio-mergex-console)

### Component: Next.js Routing Configuration

#### [MODIFY] [next.config.ts](file:///d:/web%20development/the-mergex-company/next.config.ts)
Add rewrites to proxy `/console` requests to the local/production Sanity Studio:
```typescript
import type { NextConfig } from "next";

const nextConfig: NextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'images.unsplash.com',
      },
    ],
  },
  async rewrites() {
    const isDev = process.env.NODE_ENV === 'development';
    return [
      {
        source: "/console/:path*",
        destination: isDev
          ? "http://localhost:3333/:path*"
          : "https://mergex-console.sanity.studio/:path*", // Production hosted URL
      },
    ];
  },
};

export default nextConfig;
```
